ESP-WHO 是基于乐鑫芯片的图像处理开发平台，其中包含了实际应用中可能出现的开发示例。https://github.com/espressif/esp-who/blob/master/README_CN.md
ESP32 相机项目https://robotzero.one/esp32-camera-projects/
此网站上提供了多个使用 ESP32 摄像头模块的教程，例如 ESP32-CAM、ESP-EYE 和 TTGO ESP 摄像头。为了更容易找到一个项目，我在下面列出了它们，并附有简短的摘要。有些项目只是使用相机来捕捉图像或显示视频。有些有人脸检测和识别的说明。人脸检测通常要简单得多，因为代码只检测人脸的存在。人脸识别更复杂，需要更多步骤来设置，因为必须首先对人脸进行数字映射并保存到模块中，然后才能使用它来“记住”人脸。

最容易开始和测试相机的项目是相机 Web 服务器示例，因为它已经设置好可以运行了。在Arduino IDE中，这需要10分钟，您可以测试流媒体，人脸检测和人脸识别。Arduino IDE 教程在这里： https://robotzero.one/esp32-cam-arduino-ide/ 如果您使用乐鑫 IDF，这里有一个教程： https://robotzero.one/esp32-camera-module/

如果您只想从相机录制图像而不进行面部检测或识别，另一种选择是延时教程，其中包含三个保存图像的选项： https://robotzero.one/time-lapse-esp32-cameras/

另一个简单的项目是用人脸检测来控制Sonoff。您不需要 Sonoff 开关。您可以将数据发送到另一台设备，或者只是点亮 LED。本教程在这里： https://robotzero.one/unlock-door-face-detection-sonoff/

更进一步，这是一个人脸识别教程，其中人的脸永久存储在设备上。Camera Web Server 示例不保存人脸，因此当 ESP32 重新启动时，人脸数据会丢失。本教程 https://robotzero.one/esp32-face-door-entry/ 解释了如何设置 ESP32 以保存人脸，以及如何使用人脸识别来控制门锁等设备。

教程 https://robotzero.one/esp-who-recognition-with-names/ 中介绍了一个更高级的系统，其中名称与捕获的人脸数据一起保存。完成本教程后，当使用 Blynk 识别人脸时，您可以向您的手机发送消息：https://robotzero.one/face-recognition-with-blynk/

一个完整的人脸识别访问系统，将“门禁”和“带姓名”教程结合到一个门禁系统中，您可以在其中添加和删除允许访问某个区域的列表中的人员： https://robotzero.one/access-control-with-face-recognition/

如果你喜欢和机器人一起玩，本教程有一个面部跟随自动平衡机器人：https://robotzero.one/face-tracking-robot/

如果您有 3D 打印机，则有两个用于创建相机的项目;在检测到人脸时自动拍照的自拍相机 – https://robotzero.one/esp32-cam-tft-selfie/ 和带有触摸开关的相机，可以释放快门并浏览 microSD 卡上的图像：https://robotzero.one/esp32-cam-oled-capacitive-touch-buttons/

最后，另一个相当简单的教程，仅适用于带有 PIR 运动传感器的 TTGO T-Camera： https://robotzero.one/ttgo-security-camera-pir/ 如果有人打开冰箱门，这个会通过发送照片来保护冰箱中的啤酒或巧克力！

该项目使用 Microsoft Cloud Vision API 识别和描述场景，并输出文本和语音描述 – https://robotzero.one/esp32-cam-scene-analyzer/
