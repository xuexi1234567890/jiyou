一、文字的加粗、倾斜和其它强调效果

Markdown文件中文字的书写与一般的文字编辑软件一致，直接编写内容即可，关键是如何在Markdown下进行文字的字体设置。

文字加粗： 将两个星号添加到要加粗的文字两端，如:

**加粗文字**
显示为：加粗文字
也可以通过选中要加粗的文字使用快捷键Ctrl+B。

文字斜体： 将一个星号添加到要倾斜的文字两端，如:

*倾斜文字*
显示为:倾斜文字
也可以选中要倾斜的文字使用快捷键Ctrl+I。

同时添加斜体与粗体： 如果要同时为文字添加斜体和粗体，可以在文字前后各添加三个星号，如:

***同时添加斜体与粗体效果***
显示为: *** 同时添加斜体与粗体效果*** (没效果就是不支持)
注意要在添加该效果的文字之前预留一个空格，否则不会显示任何效果。

在文字上添加删除线： 将两个波浪线添加到要加删除线的文字两端，如:

~~删除线~~
显示为:删除线
为文字添加引用效果： 在要添加引用的文字前使用 > 即可。注意，可以不在每一行都添加> ，如果引用之间没有空白行，只在第一行使用> 即可。如果引用之间存在空白行，在空白行的前端同样需要添加> 。引用行与正文之间要用空白行隔开，否则后续正文内容同样会被设置为引用。当存在嵌套引用时，对应增加 > 号即可，嵌套部分前后同样需要用空白行隔开。演示如下(知乎不支持嵌套引用，所以使用效果图片)：

引用abcfd
aaa
ddd

引用内部可以包含其它 Markdown 格式元素，但并不是所有格式都支持，需要在实际使用时测试。以上介绍的几种文字处理效果都是支持的。

注意：虽然一些时候也可以使用下划线来代替上述星号的作用，但是由于不同 Markdown 应用程序会对下划线有不同的处理方式，所以为了兼容性考虑，建议统一使用星号来强调文字。

二、文字的代码格式

Markdown 的代码格式是它的一大特色，相比于其它文字编辑软件，Markdown 可以更加快速便捷地设置一段文字为代码格式。

如果要将一段内容表示为代码，在该内容前后两端添加英文反引号即可。为了体现代码格式地不同，一下效果展示就不使用引用了：

`Hello Markdown!` 显示为：Hello Markdown!

如果要表示为代码的部分本身带有一个或多个反引号，则可以通过将要表示为代码的部分包含在双反引号内：

``Hello `Markdown`!`` 显示为：Hello `Markdown`!

当需要创建一个代码块时，可以在代码前后分别加三个反引号或者波浪线来表示一个代码块。为了方便编译器对代码进行高亮处理，最好是在代码块前的三个反引号(波浪线)之后指定代码语言，指定后Markdown会根据语言类型来针对性设置显示效果。
```python
import time
while True:
print(time.ctime())
time.sleep(5)
```
显示效果:

import time
while True:
    print(time.ctime())
    time.sleep(5)

三、Markdown中的转义字符

前面我们提到过，Markdown 通过特殊符号来表示文字的信息，比如 # 表示标题级别，* 表示加粗或斜体。那么如果我们需要在正文中表示这些特殊符号并且不希望它被识别成语法的一部分该怎么办？这时就要用到转义字符，举个例子，比如我要在正文中使用反引号，但不希望编辑器将它识别为一段代码，只需要在反引号前加一个反斜线 \ 即可。 演示效果如下：
**接下来要写一个转义字符**，`然后是另一个转义字符`
接下来要写一个转义字符，然后是另一个转义字符

四、Markdown超链接语法

1.1 插入链接： 链接是一个很广泛的概念，我们插入的图片、网址等都是以链接的形式存在，这里先对一般的超链接进行说明。在 Markdown 编辑中插入链接的语法格式如下：

[链接显示名](链接地址 "链接标题")

方括号和圆括号之间不需要隔空格。链接显示名是指该链接在预览窗口显示的内容，而链接标题是指当我们将鼠标悬停在该链接上时显示的内容，它并不是必须写的。注意链接标题要用双引号引起并且与链接地址间隔一个空格。举例如下：

[百度网站](https://www.baidu.com "百度")
显示为：百度网站

如果要插入的链接并不需要显示指定的显示名，而是直接以链接的形式显示的话，使用尖括号即可快速创建链接，举例如下：

<https://markdown.com.cn/basic-syntax/links.html>
显示为：https://markdown.com.cn/basic-syntax/links.html

注意，许多 Markdown 处理器会自动将 URL 作为链接，包括 VS Code。当我们直接书写一个网页链接时，比如：https://markdown.com.cn/extended-syntax/automatic-url-linking.html

处理器会自动将其转换成链接。如果想避免使用这一功能，可以通过将网址用反引号定义为代码来禁止:https://markdown.com.cn/extended-syntax/automatic-url-linking.html

**1.2 强调链接： **如果要给链接增加强调，比如加粗或者倾斜，需要在链接语法的整体上（即包括方括号和圆括号）加星号。而如果要给链接添加代码格式，则只需要在方括号内的链接显示名上加反引号即可，添加强调格式后的链接仍然可以作为链接使用。
本编辑器不支持语法嵌套

五、Markdown图片链接语法

2.1 插入图片： Markdown 语法下并不支持我们像其它文字编辑软件一样，直接拖动任意位置的图片即可插入，而是需要将要插入的图片提前保存在 Markdown 文件的同一目录下，使用相对路径插入(使用绝对路径时可能无法正常显示，也不利于文件的转移)。建议在编写的 Markdown 文件所在目录下新建一个专门用于存放插入图片的文件夹 Pictures 。在 VS Code 中添加图片文件到 Markdown 中有如下几种方法：

1）使用命令行插入： 在已经将图片保存在文件同一目录内的前提下，在 VS Code 中使用快捷命令 Ctrl+Shift+P 打开 VS Code 命令行，输入 Markdown : Insert Image from Workspace 命令可以快速添加一张图片到当前打开的 markdown 文件内。插入的图片以绝对路径形式显示。这种方法容易显示图片错误，而且操作起来比较繁琐，不建议使用。

2）使用拖拽插入： 如果想通过拖拽来将图片插入，只需要从 VS Code 资源管理器中或直接从图片文件夹中拖拽一个图片到 Markdown 代码区后按住 Shift 即可将它放入文件中，预览光标显示的是拖入后图片的位置。插入的图片会自动保存到当前编辑的 Markdown 文件所在目录。注意，中文命名的图片虽然可以插入，但是图片的名称会显示为乱码。

3）使用语法格式： 更多情况下我们使用特定的书写格式来导入图片，这更符合我们使用 Markdown 的初衷，也不容易出错。将图片导入Markdown的语法为：

![这是图片](https://xuexi1234567890.github.io/jiyou/post-images/1701749157400.png)
其中 Alt Text 表示插入的图片无法正常显示时图片显示的名称，之后的圆括号内书写的是图片所在的路径，建议使用相对路径，其中 / 开头的路径是相较与工作区根目录来处理的，而以 ./ 开始的路径或没有任何前缀的路径则是相较于当前文件开始处理的。
如果有图床建议使用图床或者使用网络图片

插入图片
如果需要给插入的图片添加链接的话，只需要将插入图片的语法放入方括号中，然后在方括号后增加圆括号，在圆括号内书写链接。这时如果点击图片的话，就可以跳转到图片代表的链接。

三、Markdown脚注语法
脚注本质上也是一种引用类链接，我们之前讲过引用链接的格式，脚注的书写格式与其有所不同。首先是标记名前必须有 ^ 符号，其次是可以不写前面的列表标记名：

正文[^标记名] [^标记名]:脚注内容
正文[^标记名] [^标记名]:脚注内容
脚注链接
注意，脚注显示的标号与你的标记名无关，只与其为文章的第几个脚注有关。标记名只是为了便于我们书写时理解其内容。脚注在预览中会自动添加到文章结尾。
本编辑器不支持脚注
